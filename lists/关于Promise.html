# 关于Promise(上)

# 什么是Promise？

Promise是Js关于异步编程的重要概念，是目前比较流程的Js异步解决方案

在Js当中关于处理异步的方案基本上有以下几种

  • 回调(callback)
  • 事件监听
  • 发布订阅
  • Promise
  • Async Await

# Callback Hell

设想一下，如果我们目前有一个通过读取文件内容，获得目标文件名，然后再读取内容的需求，如果我们使用回调的写法将会是怎样？

const fs = require('fs');
fs.readFile('path',(err,data)=> {
  fs.readFile(data,function()=> {
              console.log(data)
              })
})

可以看到我们的代码从竖向发展转为横向发展，而且每个请求之间相互依赖，当我们想要处理越多的异步逻辑时，就会演变成更为深的回调嵌套，这样的代码不但复用性差，当我们想要改需求的时候更是让人痛苦，根本就无法维护。

# Promise重写需求

让我们来看看如果用Promise来重写会是什么样子？

const fs = require('fs');
function getValue(file) {
    return new Promise((resolve, reject) => {
        fs.readFile(file, (err, data) => {
            if (err) {
                reject(err)
            } else {
                resolve(data)
            }
        })
    })
}

getValue('file').then((data)=> {
  return getValue(data)
}).then((data)=> {
  console.log(data)
}).catch((err)=> {
  console.log(err)
})

可以看到通过Promise封装后的代码语义化更强，维护起来更为方便。

# Promise的基础使用

Promise是一个类，我们在需要生成一个Promise实例，并给构造函数传一个函数(执行器),在执行器里面我们传入两个函数，分别为resolve和reject。通常我们还把它封装在一个函数里面，并返回。

function getValue(){
    return new Promise((resolve,reject)=> {
      
})
}

在函数体中我们可以写我们自己的异步逻辑，然后把成功的结果传入reslove函数，把错误的结果传入reject函数

const fs = require('fs');
function getValue(){
    return new Promise((resolve,reject)=> {
      re.readFile('path',(err,data)=> {
          if(err){
              reject(err)
          }else {
              resolve(data)
          }
      })
})
}

然后我们再调用Promise的then方法，在then方法中有两个参数，分别为执行成功的回调函数和失败的回调函数

getValue('test.txt').then((data)=>{
  console.log(data)
},(err)=> {
  console.log(err)
})

如果异步执行成功，那么执行的结果就会自动作为参数传入第一个回调函数，如果执行失败，关于失败的结果就会自动传入第二个回调函数。

# Promise链式调用

Promise的then方法可以提供给我们链式调用，我们只需要在函数体中把数据return出来，那么就可以传入下一个then。每一个then方法都会返回一个新的Promise。

getValue('test.txt').then((data)=>{
  return data.toString()
}).then((data)=> {
  console.log(data)
}).then(()=> {
  console.log('任务完成')
})

# Promise.catch()

catch方法实际上就是一个特殊的then方法，它帮助我们捕获在then中发送的异常。

getValue('test.txt',then(data)=> {
  return data.toString()
}).then((data)=> {
    throw new Error('出错啦')
}).catch((err)=> {
    console.log(err)
})
//Error: 出错啦
//catch方法可以捕获在他上方所有then方法中的错误。

# Promise的静态方法

  1. Promise.resolve可以返回一个状态为fulfilled的Promise对象
Promise.resolve('123').then((data)=> {
    console.log(data)
})
// 123

Promise.resolve(123)
//相当于
new Promise((resolve,reject)=> {
    resolve(123)
})
  1. Promise.reject可以返回一个状态为rejected的Promise对象
Promise.reject(new Error('出错了')).catch((err)=> {
  console.log(err)
})
// 出错了

Promise.reject(new Error('出错了'))
//相当于
new Promise((resolve,reject)=> {
  reject(new Error('出错了'))
})
  1. Promise.all,all方法是一个数组，在数组中参数是Promise实例，只要当中所有Promise状态都为filfulled，它就返回一个新的Promise,我们可以使用then方法在成功回调中获取所有的数据。
const p1 = Promise.resolve(1);
const p2 = Promise.resolve(2);
const p3 = Promise.resolve(3);

Promise.all([p1,p2,p3]).then((data)=> {
  console.log(data)
}).catch((err)=>{
    console.log(err)
})
// [1,2,3]
  1. Promise.race通用接受一个Promise对象数组作为参数，当其中一个状态转变为fulfilled或者rejected，都会立即改变返回新Promise的状态。
function timerPromise(time){
    return new Promise((resolve,reject)=> {
      setTimeOut(()=> {
        resolve(time),time)
    })
}
Promise.race([
    timerPromise(100),
    timerPromise(200),
    timerPromise(300),
      ]).then((data)=> {
          console.log(data)
      })
  // 100
