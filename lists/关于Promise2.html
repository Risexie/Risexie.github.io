# 关于Promise(下)

在上一篇文章，我们介绍了Promise主要用法和相关的方法，接下来就让我们通过源码，深入地了解一下Promise到底是如何实现的，来加深一下对Promise的理解。

事不宜迟，上源码

# Promise的三种状态

const PENDING = 'pending';
const FULFILLED = 'fulfilled';
const REJECTED = 'rejected';

function myPromise(executor) {
    let self = this;
    self.status = PENDING;
    self.onFulfilled = [];
    self.onRejected = [];

    function resolve(value) {
        if (self.status == PENDING) {
            self.status = FULFILLED;
            self.value = value
            self.onFulfilled.forEach((fn) => fn())
        }
    }

    function reject(reason) {
        if (self.status == PENDING) {
            self.status = REJECTED;
            self.reason = reason
            self.onRejected.forEach((fn) => fn())
        }
    }
    try {
        executor(resolve, reject)

    } catch (err) {
        reject(err)
    }
}

我们可以更为清晰地发现，Promise主要是有三种状态，分别为Pending，Fulfilled,Rejected

在我们New一个Promise的时候，所传进去的是一个名为Executor的函数，在Executor当中有两个分别名为resolve,reject的函数给予我们调用，而且Executor是立即执行的。

每当我们调用resolve或者reject都会修改Promise实例的状态，在调用前Promise为pending态。而分别传给resolve和reject的参数，都会分别赋值给实例的value和reason属性。然后分别调用onFulfilled和onRejected数组里面的函数，他们的作用后面会提到。

# Then方法


myPromise.prototype.then = function (onFulfilled, onRejected) {
    onFulfilled = typeof onFulfilled == 'function' ? onFulfilled : value => value;
    onRejected = typeof onRejected == 'function' ? onRejected : reason => { throw reason }
    let self = this;
    let Promise2 = new myPromise((resolve, reject) => {
        if (self.status == FULFILLED) {
            setTimeout(() => {
                try {
                    let x = onFulfilled(self.value)
                    resolvePromise(Promise2, x, resolve, reject)
                } catch (err) {
                    reject(err)
                }
            })
        } else if (self.status == REJECTED) {
            setTimeout(() => {
                try {
                    let x = onRejected(self.reason)
                    resolvePromise(Promise2, x, resolve, reject)
                } catch (err) {
                    reject(err)
                }
            })
        } else if (self.status == PENDING) {
            self.onFulfilled.push(() => {
                setTimeout(() => {
                    try {
                        let x = onFulfilled(self.value)
                        resolvePromise(Promise2, x, resolve, reject)
                    } catch (err) {
                        reject(err)
                    }
                })
            })

            self.onRejected.push(() => {
                setTimeout(() => {
                    try {
                        let x = onRejected(self.reason)
                        resolvePromise(Promise2, x, resolve, reject)
                    } catch (err) {
                        reject(err)
                    }
                })
            })

        }
    })
    return Promise2
}

接下来我们分析Promise当中的重头戏，Promise.then方法。在这之前我们就已经知道，Promise.then是支持链式调动的，所以每一个then方法返回的，肯定也是一个Promise。我们给then方法传进去的分别是名为onFulfilled和onRejected回调函数。使用一个变量self来获取旧Promise对象，并通过判断它的状态，来分别调动onFulfilled和onRejected回调，并把value和reason作为参数传入。

而且在每一个调用外都包了一层setTimeout，所以回调函数的执行都是异步的，不过这只是模拟的而已，真正的源码并不是使用这样的写法。

需要注意的是当旧Promise的状态为Pending时，我们把整个回调函数都放进了onFulfilled或onRejected当中。这其实是符合常理的，因为我们需要Promise封装的都是回调任务，通常这些任务的执行都需要花费一定的时间的，而then方法的调用是同步的，我们其实无法第一时间得知Promise的状态，所以我们只有先把回调函数以发布订阅的方式，先放进相应的数组当中，等待Promise状态的变更，才再调用。其实这里也就解释了前面凡是resolve和reject函数执行都会调用onFulfilled和onRejected数组里函数的原因。

# resolvePromise

function resolvePromise(promise2,x,resolve,reject){
  if(x === promise2){
    return reject(new TypeError('循环引用'))
  }

  if(x !== null && (typeof x === 'object' || typeof x === 'function')){

    let called
    try{
      let then = x.then
      if(typeof then === 'function'){
        then.call(x,y=>{
          if(called) return;
          called = true
          resolvePromise(promise2,y,resolve,reject)
        },err=>{
          if(called) return;
          called = true
          reject(err)
        })
      }else{
        resolve(x)
      }
    }catch(e){
      if(called) return;
      called = true
      reject(e)
    }
  }else{
    resolve(x)
  }
}

其实Promise的主要功能基本上通过上方的两段代码就已经解决了，但是我们想要把Promise继续完善，就不得不需要上方的resolvePromise函数。

可以看到在一开始就先判断x 是不是与新生成的Promise2相等，如果他们两相等，就报一个循环引用的错误。这是因为Promis2是Promise1.then方法新生成的Promise对象，promise2在一开始的时候就调用了reslovePromise函数， 有因为x是一个Promise对象，我们就需要用then方法来不断地递归它，直到它不再是一个Promise，但是promise2本身自己也在等待这个结果，就相当于它自己在等待自己，也就永远执行不下去。

然后再去判断x是不是个对象或者函数，并且如果x存在then方法，那我们就可以判断x也是一个Promise,那么我们就递归调用自身，直到不是Promise为止。如果最终得到是一个普通值，那么就直接resolve。

then.call主要是用来解决this的指向问题。

到这里，我们就基本上了解了Promise的源码，接下来我们顺便再看看Promise静态方法的实现，你会发现其实他们没有想象中的难，甚至可以说蛮简单的。

# promise静态方法

  1. Promise.catch

其实Promise.catch也就是一个特殊的then方法，它就是相当于Promise.then方法中的onRejected回调函数。

catch(onRejected){
    return this.then(null,(onRejected))
}
  1. Promise.resolve/Promise.reject

他们的作用其实就是分别返回一个状态为fulfilled和rejected的Promise

//resolve
Promise.resolve = function(val){
    return new Promise((resolve,reject)=> {
        resolve(val)
    })
}
//rejected
Promise.rejected = function(val){
    return new Promise((resolve,reject)=> {
        reject(val)
    })
}
  1. Promise.all

Promise.all()方法的参数是一个数组，里面放的都是Promise对象，当里面所有的Promise都resolve的时候，返回一个成员为所有Promise结果的数组，并把结果包装成一个新的Promise实例，也就是传入新Promise的会调函数。如果当中一旦与任何一个Promise状态为rejected，那么就将返回值传递给新Promise实例的错误回调。

Promise.all = function(promises){
    let arr = [];
    let i = 0;
  function processData(index,data){
    arr[index] = data
    i++
    if(i = promises.length){
        resolve(arr)
    }
  }
  return new Promise((resolve,reject)=> {
    for(let i = 0;i < promises.length;i ++){
        arr[i].then((data)=> {
          processData(i,data)
        },reject)
    }
  })
}
  1. Promise.race

Promise.race方法同样是以多个Promise实例为参数，然后把结果保证成一个新的Promise实例，不同的当中只要有一个Promise实例的状态率先改变，新Promise实例的状态就跟着改变，并以率先改变的Promise返回值作为参数，传入新Promise的回调函数。
